Git — это распределенная система управления версиями, предназначенная для отслеживания изменений в файлах (преимущественно коде), сохранения истории правок, безопасных экспериментов и совместной работы команд разработчиков. Она позволяет фиксировать промежуточные состояния проекта («коммиты»), легко возвращаться к ранним версиям и объединять работу нескольких программистов. 
Основные задачи и функции Git:
Контроль версий и история: Git сохраняет историю всех изменений в проекте. В любой момент можно посмотреть, кто, что и когда изменил.
Отмена изменений: Если новый код содержит ошибку или перестал быть нужным, Git позволяет быстро откатиться к предыдущей рабочей версии.
Командная работа (ветвление): Разработчики могут создавать собственные ветки (branches) для создания новых функций, не влияя на основную рабочую версию проекта.
Слияние (Merging): По завершении работы над фичей, ветку можно объединить с основной, при этом Git помогает разрешать конфликты, если над одним файлом трудились несколько человек.
Безопасность и хранение: Даже если локальные файлы будут удалены, историю можно восстановить из репозитория (удаленного сервера, например, на GitHub или GitLab).
